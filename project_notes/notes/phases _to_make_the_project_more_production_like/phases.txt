The roadmap I'd recommend for your project

Given your goal of preparing for backend interviews, I'd tackle the remaining work in this order:

Refactor resolver files (one file per GraphQL type).
Implement a proper subscription manager (broadcast to multiple subscribers).
Add DataLoaders to solve the N+1 problem.
Introduce authentication and authorization with JWT and context.
Add transactions where operations span multiple repositories.
Implement structured error handling with GraphQL error extensions.
Add middleware (logging, auth, recovery, metrics).
Write comprehensive tests for repositories, services, and resolvers.
Dockerize the application with PostgreSQL and add docker-compose.
Add observability (structured logging, tracing, metrics).

Phase 4 — Real Subscription Manager ⭐⭐⭐⭐⭐

Right now you have

chan *User

Production systems don't.

Instead

type SubscriptionManager struct {

    userSubscribers map[string]chan *User

    postSubscribers map[string]chan *Post

    commentSubscribers map[string]chan *Comment

    mutex sync.RWMutex
}

Now

CreateUser()

broadcasts to every subscriber.
